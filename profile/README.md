## 📏 Gihhub 사용 규칙

### 1. 브랜치 네이밍 규칙
| 타입 | 설명 | 예시 |
|------|------|------|
| `feat/` | 새로운 기능 개발 | `feat/group-order-api` |
| `fix/` | 버그 수정 | `fix/order-time-bug` |
| `refactor/` | 코드 리팩토링 | `refactor/menu-component` |
| `hotfix/` | 긴급 수정 | `hotfix/order-crash` |

---

### 2. 커밋 메시지 규칙
```bash
<type>(<scope>): <message>

# type 예시
feat     : 새로운 기능 추가
fix      : 버그 수정
refactor : 코드 구조 변경
docs     : 문서 수정
style    : 포맷팅, 세미콜론 누락 등
test     : 테스트 코드 추가/수정

```
